# Language Support

## MVP Targets

- JavaScript
- TypeScript

## Why These Languages First

The initial WatchLLM write-path risk surface is concentrated in JavaScript and
TypeScript codebases used by agent-assisted application development and editor
workflows. Focusing on these targets keeps the kernel narrow, testable, and
measurable during MVP.

## Out of Scope for Current Kernel Phase

Unsupported languages are intentionally out of scope for now. Expanding language
coverage is a post-MVP step and must follow the same deterministic contracts,
fixture discipline, and benchmark requirements as existing support.
